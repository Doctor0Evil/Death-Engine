# Death-Engine BCI Runtime Layout and Lifecycle

This document defines the target directory tree for Death-Engine’s BCI components, the promotion lifecycle for HorrorMappingConfig.BCI bindings from `lab` to `standard` and `mature`, and the strategy for schema evolution, feature gating, and input-domain constraints.

The intent is to keep Death-Engine aligned with the HorrorPlace-Constellation-Contracts BCI spine, including `bci-feature-envelope-v2.json`, `bci-metrics-envelope-v2.json`, `bci-core-intensity-envelope-v1.json`, `HorrorMappingConfig.BCI.v1`, and the four CI gates defined in the constellation.
